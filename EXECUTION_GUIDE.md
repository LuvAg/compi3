
## What Each Part Demonstrates

### Part 1 & 2: Conflict Analysis
When you run the parser, it outputs detailed analysis of:
- LALR(1) automaton construction (26 states)
- **Conflict #1: Dangling Else** - Resolved by explicit grammar rules
- **Conflict #2: Operator Precedence** - Resolved by precedence declarations
- **Conflict #3: Reduce/Reduce** - None present (unique first terminals)

### Part 3: Reverse Derivation Tree
The parser displays the parse tree in **reverse (post-order)** order:
- Shows leaves (terminals) first
- Internal nodes (non-terminals) next
- Root (program) last
- This order matches how the shift-reduce parser builds the tree bottom-up

### Part 4: Parsing Table
Complete LALR(1) parsing table showing:
- All 45 production rules
- Action table entries (shift/reduce/accept)
- Goto table entries for non-terminals
- Full notation explanation

### Part 5: Error Diagnostics
Comprehensive error detection for:
- Syntax errors (unexpected tokens)
- Structural errors (missing markers, unclosed blocks)
- Declaration errors (invalid types, missing names)
- Expression errors (unmatched parentheses)
- Statement errors (missing conditions/bodies)

### Part 6: Additional Information
Implementation details including:
- Architecture (Lexer, Parser, Utilities)
- Performance characteristics
- Build system explanation
- Testing infrastructure

## Complete Grammar Reference

### Variable Declaration
```brolang
bro num x = 42;           // variable with initializer
bro str name = "John";    // string variable
bro bool flag = sahi;     // boolean variable
pakka bro num PI = 3.14;  // constant declaration
```

### Control Flow
```brolang
agarbro (x > 5) {         // if statement
  bolbro "greater";
}
naitobro (x == 5) {       // else if
  bolbro "equal";
}
warnabro {                // else
  bolbro "less";
}
```

### Loops
```brolang
jabtakbro (i < 10) {      // while loop
  bolbro i;
  baskarbro;              // break
  agladehbro;             // continue
}
```

### Expressions
```brolang
bro num sum = 2 + 3 * 4;  // 14 (precedence: * before +)
bro num result = (2 + 3) * 4;  // 20 (force order with parens)
bro bool cond = (x > 5) == sahi;  // comparison
```

## Compilation Details

### Lexer Generation
```bash
flex -o lex.yy.c brolang_lexer.l
gcc -Wall -Wextra -g -O2 -c -o lex.yy.o lex.yy.c
```

### Parser Generation
```bash
bison -d -o brolang_parser.tab.c brolang_parser.y
gcc -Wall -Wextra -g -O2 -c -o brolang_parser.tab.o brolang_parser.tab.c
```

### Linking
```bash
gcc -Wall -Wextra -g -O2 -o brolang_parser lex.yy.o brolang_parser.tab.o -lm
```

### View Complete Parser Automaton
```bash
bison -v brolang_parser.y
cat brolang_parser.output
```

This generates a detailed report of all 26 LALR(1) states with their item sets and transitions.

## Troubleshooting

**Q: Parser won't compile**
A: Ensure `bison` and `flex` are installed:
```bash
sudo apt-get install bison flex
```

**Q: Can't find input file**
A: Provide full or relative path:
```bash
./brolang_parser ./testcases/valid/test1_basic.bro
```

**Q: Error message is unclear**
A: Full error output with line numbers is provided. Check [PART 5] for error type catalog.

**Q: Want to see detailed LALR(1) states**
A: Use: `bison -v brolang_parser.y && cat brolang_parser.output`
